# Pricing Sections

## Purpose

Pricing sections:
- Present options clearly
- Guide toward preferred plan
- Build trust
- Reduce friction to purchase

## Highlighting the Preferred Plan

### Techniques

1. **Badge:** "POPULAR", "BEST VALUE", "RECOMMENDED"
2. **Scale:** Make card larger/taller
3. **Border:** Add primary color border
4. **Background:** Different background color
5. **Position:** Center position (Z-pattern focus)
6. **CTA style:** Filled vs outlined for others

### Example

```
┌───────────┐  ╔═══════════════╗  ┌───────────┐
│  Starter  │  ║     Pro       ║  │Enterprise │
│           │  ║   POPULAR     ║  │           │
│   $0      │  ║    $29/mo     ║  │  $99/mo   │
│           │  ║               ║  │           │
│ [Start]   │  ║ [Get Started] ║  │ [Contact] │
│  outline  │  ║    filled     ║  │  outline  │
└───────────┘  ╚═══════════════╝  └───────────┘
                      ↑
              Larger, bordered, filled CTA
```

## Pricing Psychology

### Charm Pricing

```
❌ $30     →  ✅ $29
❌ $100    →  ✅ $99
❌ $10     →  ✅ $9.99
```

Left digit effect: $29 feels significantly less than $30.

### Precise Numbers

```
Standard:     $250/mo
Precise:      $247/mo  ← Feels calculated, trustworthy
```

Precise prices suggest careful consideration.

### Discount Framing

| Price Point | Better Discount Display |
|-------------|------------------------|
| Under $100 | Percentage: "50% OFF" |
| Over $100 | Dollar amount: "$200 OFF" |

```
$50 product:  "50% OFF" feels bigger than "$25 OFF"
$1000 product: "$200 OFF" feels bigger than "20% OFF"
```

### Anchoring

Show original price to anchor value:

```
$99  $79/month
 ↑      ↑
Strike  Current
through
```

### Decoy Effect

Add a plan that makes target plan look better:

```
Basic: $10 (5 features)
Pro:   $25 (15 features)  ← Target (great value vs Plus)
Plus:  $30 (17 features)  ← Decoy (only 2 more features for $5)
```

## Risk Reducers

### Examples

```
✓ 14-day free trial
✓ No credit card required
✓ Cancel anytime
✓ 30-day money-back guarantee
✓ Secure checkout
```

### Placement

- Below CTA button
- As feature in list
- Footer of pricing section

## Feature Lists

### Do

- Start with most valuable features
- Use clear, benefit-focused language
- Keep consistent across plans
- Group related features
- Show what's included AND excluded

### Don't

- Use jargon
- List too many features (cognitive overload)
- Hide important limitations
- Use vague descriptions

### Feature Display

```
Included:
✓ Unlimited projects
✓ 10 team members
✓ Priority support

Not included:
✕ Custom integrations    ← Show what they don't get
✕ Dedicated account manager
```

## Common Mistakes

| Mistake | Fix |
|---------|-----|
| All plans look equal | Highlight recommended plan |
| Too many plans | 3-4 maximum |
| Feature overload | Show top 5-7, link to full list |
| No risk reducers | Add trial, guarantee, cancel anytime |
| Hidden pricing | Be transparent, no surprises |
| Confusing tiers | Clear value progression |
| No social proof | Add logos, reviews, testimonials |
| Yes/No for actions | Use descriptive CTA text |
